# Repository Structure

Each circuit exported from IBM Quantum Composer contains two types of files:

### 1. OpenQASM Files (`.qasm`)

These files contain the **text representation of the quantum circuit** written in **OpenQASM (Open Quantum Assembly Language)**.

OpenQASM describes:

- quantum registers
- classical registers
- quantum gates
- measurements

Example structure inside a `.qasm` file:

```
qreg q[2];
creg c[2];

h q[0];
cx q[0], q[1];

measure q[0] -> c[0];
measure q[1] -> c[1];
```

These files can be directly executed using **Qiskit** or other quantum simulators.

---

### 2. Circuit Diagram Files (`.svg`)

Each `.svg` file is a **visual representation of the quantum circuit** generated by IBM Quantum Composer.

The diagram shows:

- qubit wires
- quantum gates (H, X, Z, CNOT, etc.)
- measurement operations
- classical registers

SVG files are vector graphics, which means they can be used in:

- reports
- presentations
- research documentation
- GitHub READMEs

without losing resolution.

# Quantum Concepts Demonstrated

The circuits in this repository illustrate several fundamental ideas in quantum computing:

### Quantum Superposition

Superposition is created using the **Hadamard gate (H)**

$H|0\rangle = \frac{|0\rangle + |1\rangle}{\sqrt{2}}$

This allows a qubit to exist in multiple states simultaneously.

---

### Quantum Entanglement

Entanglement is created using a combination of **Hadamard and CNOT gates**.

Example Bell state:

$|\Phi^+\rangle = \frac{|00\rangle + |11\rangle}{\sqrt{2}}$


Entangled states are fundamental to many quantum protocols such as:

- quantum teleportation
- superdense coding
- quantum communication

---

### Quantum Measurement

Quantum measurements collapse a qubit state into classical outcomes.

Measurement is represented as:

```
measure q[i] -> c[i]
```

where the quantum state is mapped to a classical bit.

# Running the Circuits in Qiskit

The `.qasm` files can be loaded and executed using **Qiskit**.

Example Python code:

```python
from qiskit import QuantumCircuit
from qiskit import Aer, execute

qc = QuantumCircuit.from_qasm_file("circuit.qasm")

backend = Aer.get_backend('qasm_simulator')
result = execute(qc, backend, shots=1024).result()

print(result.get_counts())
```

This allows the circuits to be simulated or run on real IBM quantum hardware.
